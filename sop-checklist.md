# 📋 Standard Operating Procedures (SOPs)

**Purpose:** Document all workflows so the agency can run without you  
**Owner:** Project Manager (updates quarterly)  
**Review:** First Monday of each month

---

## 🎬 VIDEO PRODUCTION WORKFLOW

### Overview
```
Brief Received → Script Approved → Filming → Editing → QA → Delivery → Feedback → Archive
     │              │                │           │         │       │          │
   PM             PM/You          Creator     Editor    PM      PM        Creator
```

---

### SOP 1: BRIEF INTAKE

**Owner:** Project Manager  
**Time:** 15-30 min per brief  
**Tools:** Google Form → Notion/Trello → WhatsApp

#### Steps
1. **Receive brief from client** (WhatsApp/email/form)
   - Video topic/theme
   - Target platform (Reels, TikTok, YouTube Shorts)
   - Key message/CTA
   - Deadline
   - Any references/examples

2. **Log in project tracker**
   - Create card in Trello/Notion
   - Assign unique ID (e.g., VID-2026-001)
   - Tag: `pending-script`, `filming`, `editing`, `qa`, `ready`

3. **Confirm with client**
   - Repeat back requirements
   - Confirm deadline
   - Flag any concerns (tight timeline, unclear brief)

4. **Assign to Content Creator**
   - Notify via WhatsApp/Slack
   - Include brief + deadline
   - Script due: 48 hours from brief receipt

#### Template: Brief Confirmation Message
```
Hi [Client Name]! 👋

Got your brief for [topic]. Just to confirm:
- Platform: [Reels/TikTok/Shorts]
- Key message: [message]
- CTA: [call-to-action]
- Deadline: [date]

We'll send the script for approval by [date + 2 days]. Sound good?

— [PM Name]
```

### SOP 2: SCRIPTWRITING

**Owner:** Content Creator  
**Time:** 30-45 min per script  
**Tools:** Google Docs → Client Approval → Notion

#### Steps
1. **Review brief**
   - Understand target audience
   - Identify hook (first 3 seconds)
   - Define CTA (call-to-action)

2. **Write script** (30-sec video = 75-90 words)
   - Hook (0-3 sec): Grab attention
   - Problem (3-10 sec): Relatable pain point
   - Solution (10-25 sec): Product/service benefit
   - CTA (25-30 sec): What to do next

3. **Format for client review**
   - Google Doc with clear sections
   - Include visual notes in [brackets]
   - Add timing markers (0-3s, 3-10s, etc.)

4. **Send to PM for client approval**
   - Allow 24-48 hours for client feedback
   - Track revision round # (max 2 included)

5. **Finalize script**
   - Incorporate approved changes
   - Mark as `ready-to-film` in tracker

#### Template: 30-Second Script Format
```
VIDEO ID: VID-2026-001
TOPIC: [Topic]
PLATFORM: [Reels/TikTok/Shorts]
DURATION: 30 seconds
WORD COUNT: 85 words

---

[0-3s] HOOK
Visual: [Description]
Audio: "Did you know [surprising fact]?"

[3-10s] PROBLEM
Visual: [Description]
Audio: "Most people struggle with [pain point]..."

[10-25s] SOLUTION
Visual: [Description]
Audio: "That's why we created [product]. Here's how it works..."

[25-30s] CTA
Visual: [Description]
Audio: "Click the link in bio to [action]!"

---
NOTES:
- Background music: Upbeat, corporate
- Text overlay: Bold, brand colors
- End card: Logo + website URL
```

### SOP 3: FILMING

**Owner:** Content Creator  
**Time:** 2-3 hours per batch (10-15 videos)  
**Tools:** Camera, lighting, mic, teleprompter app

#### Steps
1. **Pre-production checklist** (15 min before filming)
   - [ ] Camera battery charged (100%)
   - [ ] SD card formatted, 50GB+ free
   - [ ] Lighting set up (key + fill + beauty)
   - [ ] Audio test (Rode Wireless Go II)
   - [ ] Teleprompter app loaded with scripts
   - [ ] Wardrobe check (solid colors, no patterns)
   - [ ] Background tidy, branded if possible

2. **Filming session** (2-3 hours for 10-15 videos)
   - Shoot in batches (group by topic/wardrobe)
   - 3 takes per video (best one selected later)
   - Slate each take (hold up card with video ID)
   - Review footage every 5 videos (quality check)

3. **Post-filming workflow**
   - Transfer footage to editing workstation
   - Organize by video ID (folder structure below)
   - Backup to cloud (Google Drive/Dropbox)
   - Mark as `ready-to-edit` in tracker

#### Folder Structure
```
📁 RAW FOOTAGE/
  └── 📁 2026-06/
      └── 📁 VID-2026-001/
          ├── 📄 VID-2026-001_Take1.mp4
          ├── 📄 VID-2026-001_Take2.mp4
          ├── 📄 VID-2026-001_Take3.mp4
          └── 📄 VID-2026-001_Script.pdf
```

### SOP 4: VIDEO EDITING

**Owner:** Video Editor  
**Time:** 60-90 min per video (medium complexity)  
**Tools:** Adobe Premiere Pro / DaVinci Resolve / CapCut

#### Steps
1. **Select best take**
   - Review all 3 takes
   - Choose: Best audio + best framing + best delivery
   - Delete other takes (or archive)

2. **Rough cut** (15-20 min)
   - Trim to 30 seconds
   - Remove dead air, pauses, mistakes
   - Add J-cuts/L-cuts for smooth transitions

3. **Add elements** (30-40 min)
   - [ ] Color correction (exposure, contrast, saturation)
   - [ ] Audio leveling (normalize to -16 LUFS)
   - [ ] Background music (lower during dialogue)
   - [ ] Captions (auto-generate + manual correction)
   - [ ] B-roll inserts (if applicable)
   - [ ] Transitions (cut, fade, or branded transition)
   - [ ] End card (logo + CTA)

4. **Export** (10 min)
   - Format: MP4 (H.264)
   - Resolution: 1080x1920 (9:16 vertical)
   - Frame rate: 30 fps
   - Bitrate: 15-20 Mbps

5. **Upload for QA**
   - Save to `PENDING QA` folder
   - Update tracker: `ready-for-qa`
   - Notify PM via WhatsApp/Slack

#### Quality Checklist (Editor Self-Check)
- [ ] Audio clear, no background noise
- [ ] Video stable, no accidental camera shake
- [ ] Captions 100% accurate, synced properly
- [ ] Branding consistent (colors, fonts, logo)
- [ ] No black frames or glitches
- [ ] Export settings correct for platform

### SOP 5: QUALITY ASSURANCE

**Owner:** Project Manager  
**Time:** 10-15 min per video  
**Tools:** QA Checklist, Notion/Trello

#### Steps
1. **Review video** (10 min)
   - Watch full video at 1x speed
   - Check against QA checklist (below)
   - Note any issues with timestamps

2. **Decision: Approve or Revise**
   - **Approve:** Move to `ready-for-delivery`
   - **Revise:** Send back to editor with notes

3. **If revision needed:**
   - Document specific changes (timestamp + description)
   - Assign revision round # (track: 1/2, 2/2)
   - Set 24-hour turnaround expectation
   - Notify editor via WhatsApp/Slack

#### QA Checklist
```
VIDEO ID: VID-2026-001
REVIEWER: [PM Name]
DATE: [Date]

CONTENT:
[ ] Hook is engaging (first 3 seconds)
[ ] Message is clear and concise
[ ] CTA is strong and specific
[ ] No factual errors or typos

TECHNICAL:
[ ] Audio clear, no background noise
[ ] Video stable, good framing
[ ] Lighting consistent, no harsh shadows
[ ] Color grading looks professional

BRANDING:
[ ] Captions accurate and synced
[ ] Logo present (end card or watermark)
[ ] Colors match brand guidelines
[ ] Fonts are on-brand

PLATFORM:
[ ] Correct aspect ratio (9:16 vertical)
[ ] Correct duration (30 sec ±2 sec)
[ ] Correct format (MP4, H.264)
[ ] File named correctly (VID-2026-001_Reels.mp4)

REVISION ROUND: [1/2 or 2/2]
STATUS: [Approved / Revision Needed]

NOTES:
[List any changes needed with timestamps]
```

### SOP 6: CLIENT DELIVERY

**Owner:** Project Manager  
**Time:** 15-30 min per batch  
**Tools:** Google Drive/Dropbox, WhatsApp, Email

#### Steps
1. **Prepare delivery package**
   - Organize approved videos by platform
   - Create platform-specific folders (Reels, TikTok, Shorts)
   - Include caption suggestions (optional)
   - Add posting schedule recommendations

2. **Upload to shared folder**
   - Google Drive/Dropbox link
   - Folder naming: `YYYY-MM-DD_Delivery_[ClientName]`
   - Set permissions: Client can view/download

3. **Send delivery notification**
   - WhatsApp + Email (redundant)
   - Include: Number of videos, platforms, deadline to review
   - Request confirmation of receipt

4. **Track feedback window** (48 hours)
   - Day 1: Delivery sent
   - Day 2: Follow up if no response
   - Day 3: Assume approved if no feedback (per contract)

#### Template: Delivery Message
```
Hi [Client Name]! 🎬

Your [number] videos for [month/week] are ready!

📁 View/Download: [Google Drive link]

📋 DETAILS:
- Total videos: [number]
- Platforms: [Reels, TikTok, Shorts]
- Recommended posting: [dates/times]

⏰ FEEDBACK DEADLINE: [Date + 48 hours]
If we don't hear back by [date], we'll consider these approved and move to the next batch.

Questions? Just reply here! 😊

— [PM Name]
```

### SOP 7: FEEDBACK & REVISIONS

**Owner:** Project Manager  
**Time:** 15-30 min per revision round  
**Tools:** Notion/Trello, WhatsApp

#### Steps
1. **Collect client feedback**
   - Consolidate all notes into single document
   - Clarify vague feedback ("make it pop" → specific request)
   - Confirm revision round # (1/2 or 2/2)

2. **Send to editor**
   - Include video IDs needing revision
   - Timestamped notes for each change
   - Deadline: 24-48 hours

3. **Editor completes revisions**
   - Follows SOP 4 (editing workflow)
   - Marks as `ready-for-qa-revision`

4. **PM re-QAs revised videos**
   - Follows SOP 5 (QA checklist)
   - If approved: Re-deliver to client
   - If still issues: Escalate to you (rare)

5. **Final delivery**
   - Mark video as `complete` in tracker
   - Archive all files
   - Log revision count (for monthly report)

#### Revision Policy (Contract Terms)
- **Included:** 2 revision rounds per video
- **Round 1:** Major changes (timing, messaging, structure)
- **Round 2:** Minor tweaks (captions, color, music level)
- **Round 3+:** Billed at RM 150/video (change order required)

### SOP 8: ARCHIVING

**Owner:** Content Creator  
**Time:** 10 min per batch  
**Tools:** Google Drive, External HDD

#### Steps
1. **Organize completed project**
   - Move all files to `ARCHIVE/YYYY/MM/`
   - Include: Raw footage, scripts, final exports, feedback

2. **Backup to cloud**
   - Google Drive/Dropbox (auto-sync)
   - Verify upload complete

3. **Backup to external drive**
   - Copy to 2TB external HDD (weekly)
   - Store offsite (your home or safety deposit box)

4. **Update tracker**
   - Mark project as `archived`
   - Include archive location path

#### Archive Structure
```
📁 ARCHIVE/
  └── 📁 2026/
      └── 📁 06-June/
          └── 📁 VID-2026-001/
              ├── 📁 01_Script/
              ├── 📁 02_RawFootage/
              ├── 📁 03_Edits/
              ├── 📁 04_FinalExports/
              ├── 📁 05_Feedback/
              └── 📄 ProjectSummary.pdf
```

## 📊 WEEKLY/MONTHLY RITUALS

### Weekly Team Standup (Monday, 30 min)
**Attendees:** You, PM, Creator, Editor(s)  
**Agenda:**
1. Review last week's delivery (what went well, what didn't)
2. This week's targets (videos to deliver, deadlines)
3. Blockers (anything slowing the team down)
4. Wins (celebrate something, even small)

### Monthly Client Report (First Friday)
**Owner:** PM  
**Content:**
- Videos delivered (count + links)
- Average revision rounds
- Top-performing videos (if analytics available)
- Upcoming month's content themes
- Any issues + how we're fixing them

### Monthly Risk Review (First Monday)
**Owner:** PM (you attend)  
**Content:**
- Cash balance vs. burn rate
- Client satisfaction score
- Team morale check
- Equipment status
- Pipeline review
- Update risk matrix
